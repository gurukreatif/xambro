Konfigurasi & Build APK

Atur tampilan, PIN pengawas, dan identitas aplikasi — lalu klik Build APK untuk otomatis generate APK via GitHub Actions.

🔑

Koneksi GitHub

Token & repo untuk trigger GitHub Actions workflow

Buat di GitHub → Settings → Developer settings → Personal access tokens (classic). Scope yang dibutuhkan: repo (untuk private repo) atau public_repo (untuk public). Token disimpan di browser Anda saja, tidak dikirim ke server lain.

Nama file workflow di .github/workflows/. Default: build.yml

Nama & Deskripsi

Teks yang muncul di header layar utama Android

Teks besar di dalam header aplikasi. Bisa sama dengan nama di atas.

Teks kecil di bawah judul. Maks. 60 karakter.

Pengaturan Keamanan

PIN pengawas dan identitas package Android

Lemah

PIN yang harus dimasukkan pengawas untuk keluar dari mode ujian. Jangan gunakan 1234!

Format: id.namaorganisasi.namaapp — hanya huruf kecil, angka, dan titik. Ini adalah identitas unik APK di perangkat.

Format semantic versioning: MAJOR.MINOR.PATCH

Export Konfigurasi Manual

File JSON ini bisa dihosting dan dibaca oleh aplikasi saat startup untuk konfigurasi dinamis tanpa rebuild APK.

ℹ️ Cara Kerja Builder
  1. Isi token GitHub & nama repo
  2. Atur nama, warna, dan PIN
  3. Klik Build APK
  4. Tunggu ~3 menit
  5. Unduh APK dari link Artifacts
⚠️ Pastikan file .github/workflows/build.yml dan scripts/apply_config.py sudah ada di repo GitHub Anda.
